# Project: M5-LJEC — Limb Joints and End Caps

**Description:** Add multi-segment limb rigs to the humanoid rig model, procedural ball-joint visuals, configurable end-cap primitives, and backward-compatible bone naming. Implemented entirely in Python/Blender asset generation — not Godot. Tests live under `asset_generation/python/tests/`.

## Notes

- All implementation is Python only — no Godot/GDScript changes.
- Test runner: `uv run pytest asset_generation/python/tests/ -q` from the repo root. The existing `conftest.py` mocks `mathutils` and `bpy`; new tests must be compatible with that mock setup.
- `limb_chain.py` is pure logic (Vector arithmetic, BoneSpec construction) — fully testable without Blender.
- Mesh helper functions call `create_cylinder`, `create_sphere`, `create_box` from `blender_utils.py` and therefore require bpy at runtime; in tests they must be mocked or left to manual verification.
- Task 4 and Task 5 can be executed in the same agent run since both are Core Simulation (pure logic) and Task 5 depends directly on Task 4's output.
- Task 6 and Task 7 can be executed in the same agent run for efficiency; both are Engine Integration and Task 7 directly follows Task 6.
- Quadruped extension (`QuadrupedSimpleRig`) is explicitly out of scope for v1 per the ticket design decisions.
- The `ARM_SEGMENTS` / `LEG_SEGMENTS` ClassVar[int] fields are expected to auto-surface in the frontend `BuildControls` panel through existing ClassVar[int] introspection — NO frontend task is needed.
