## Resolutions

1. **Test naming:** Files by subsystem + behavior (e.g., `test_governance_check.py`); no ticket numbers in filenames.
2. **Isolation:** Use `unittest.mock.patch` for subprocess/file I/O; avoid mocking internal gate logic.
3. **Determinism:** All tests use isolated fixtures, no shared state or filesystem mutation outside tmp_path.
4. **Documentation:** Each test class documents which spec rules it covers (e.g., "Covers AR-01, AR-02, AR-03").
5. **Traceability:** Module docstring references spec and ticket; test classes reference spec rule ids in comments.

1. **Behavioral Design:** Each test validates observable behavior (code samples, JSON outputs, file handling) rather than gate implementation details.

6. **Edge Cases:** Handling of missing files, empty codebases, malformed configs, concurrent executions, performance on large codebases.

7. **Integration:** Tests verify gate registration, gate_runner invocation, and Taskfile task presence.
