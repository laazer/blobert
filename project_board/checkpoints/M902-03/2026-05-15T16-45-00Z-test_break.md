## Key Findings & Gaps Exposed

### 1. **Rule Evasion Gaps**

**Finding:** Several evasion techniques may not be caught by initial semprep rules:
- **Reflection via `inspect` library:** RF-01 targets `getattr`/`setattr`; `inspect.getmembers()` may bypass.
- **Silent exception via `contextlib.suppress()`:** EX-02 targets direct `except:` patterns; context manager indirection may bypass.
- **Lazy conditional logging:** OB-02 detects logger presence; lazy evaluation (condition that's rarely true) passes static check but fails intent.

**Recommendation for Task 8:** Expand semprep patterns to cover:
- Generic introspection libraries (inspect, importlib.inspect, etc.)
- Context manager exception suppression
- Semantic logging checks (must log on all exception paths, not just sometimes)

### 5. **Governance Bypass Detection Gaps**

**Finding:** High-confidence patterns (--no-verify, blanket disable) are easy to detect; semantic bypasses are hard:
- Direct tool invocation (grep for semprep/eslint/ruff commands) ✓ DETECTABLE
- Conditional skip (grep for if/SKIP/conditional) ✓ MEDIUM confidence
- Environment variable bypass (requires semantic analysis) ✗ HARD
- Modified gate runner (requires code signing) ✗ VERY HARD
- Justification comments (intentional silent exceptions) ✗ SEMANTIC

**Recommendation for Task 8:** Implement in phases:
1. **Phase 1 (MVP):** High-confidence grep patterns (--no-verify, blanket disable, direct tool call)
2. **Phase 2 (M903):** Conditional skip detection (heuristic)
3. **Phase 3 (Future):** Semantic bypass detection (code signing, comprehensive logging)

### 6. **False Positive Prevention Gaps**

**Finding:** Tests expose risk of false positives on patterns in comments/strings:
- Bare except in docstring: `"""Don't use bare except:"""` should NOT flag
- Violation code in multiline string example: `"""except: pass"""` should NOT flag
- Suppression comment in string: should NOT be interpreted as actual suppression

**Recommendation for Task 8:** Verify semprep context awareness:
- Ensure semprep does NOT flag code in string literals
- Ensure semprep does NOT flag code in comments (unless target is comments)
- Test with real-world examples (documentation, examples, etc.)
