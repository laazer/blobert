## REMAINING GAPS & VULNERABILITIES

### Gap 1: Configuration Validation
- **Issue:** No tests validate that required config files exist and have correct structure.
- **Exposure:** Missing configs silently use defaults; gate produces no output or crashes.
- **Mitigation:** Implementation must check config existence and validity before tool invocation.
- **Test:** `test_pyproject_with_empty_dev_deps_list` and friends.

### Gap 2: Tool Availability Checking
- **Issue:** Gate may not check if tools are installed before running them.
- **Exposure:** Gate crashes with "command not found" for missing tools; no graceful skip.
- **Mitigation:** Implement tool availability check using `shutil.which()` before invocation.
- **Test:** `test_assume_all_tools_present_challenges_it`.

### Gap 3: JSON Schema Validation
- **Issue:** Output JSON may not conform to gate schema; missing or wrong fields.
- **Exposure:** Downstream gates fail; checkpoint agent cannot parse violations.
- **Mitigation:** Gate must validate output against schema before returning.
- **Test:** `test_gate_output_missing_required_fields`, `test_gate_output_with_invalid_severity_values`.

### Gap 4: Tool Output Parsing Robustness
- **Issue:** Tool output parsing assumes perfect JSON; doesn't handle mixed text or malformed JSON.
- **Exposure:** Gate crashes on malformed output; tools with stderr output ignored.
- **Mitigation:** Implement robust parsing with fallback and graceful error handling.
- **Test:** `test_tool_with_mixed_json_and_text`, `test_tool_with_only_stderr_output`.

### Gap 5: Reproducibility
- **Issue:** Lock files may not be truly deterministic; floating point precision or key ordering varies.
- **Exposure:** CI runs diverge; hashes/signatures don't match across runs.
- **Mitigation:** Ensure lock files are generated with consistent serialization and no timestamps.
- **Test:** `test_lock_file_with_floating_point_precision`.

### Gap 6: Boundary Value Handling
- **Issue:** Boundary values (0, -1, huge numbers) may crash or produce wrong results.
- **Exposure:** Edge-case violations crash gate or are silently ignored.
- **Mitigation:** Validate and normalize boundary values; handle gracefully.
- **Test:** `test_line_number_zero`, `test_empty_file_path`, `test_jscpd_config_with_huge_threshold`.
