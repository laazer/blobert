## Gaps Identified (Why Test Break Phase is Needed)

### Gap 1: Type Mutation & Discriminated Union Handling
**Current Coverage:** Tests pass correct ColorPickerValue objects.  
**Missing:** Tests that pass wrong types, partial objects, or mismatched discriminators.  
- What if onChange receives `{ type: "single" }` (missing color)?  
- What if it receives `{ type: "gradient", colorA: "..." }` (missing colorB)?  
- What if HexStrControlRow tries to access v.colorA when v.type === "single"?

**Action:** Add strict type narrowing tests with malformed payloads.

### Gap 2: Regex Pattern Robustness for Hex Detection
**Current Coverage:** `_hex` and `_texture_*_color` patterns are tested for normal cases.  
**Missing:** Edge cases like:
- `_hex_` (double underscore or suffix variation)
- `Hex` (capitalization variation)
- `_HEX` (all caps)
- `_h_e_x_` (with interspersed underscores)
- Keys that contain `hex` but don't end/match the pattern
- Keys that contain `texture` and `color` but not in the expected order

**Action:** Add mutation tests with regex variations and case sensitivity checks.

### Gap 3: Hex Value Parsing Edge Cases
**Current Coverage:** Valid hex (6 chars), invalid (too short, not hex, garbage).  
**Missing:**
- Null/undefined clipboard content
- Empty string clipboard
- Clipboard with only `#` (no digits)
- Very long clipboard values (> 1000 chars)
- Clipboard with Unicode/emoji
- Concurrent paste requests (debounce behavior)
- Clipboard read timeout/rejection (Promise rejection)

**Action:** Add stress and error-handling tests for clipboard integration.

### Gap 4: Direction Value Normalization
**Current Coverage:** Valid directions (horizontal, vertical, radial) pass through.  
**Missing:**
- Null/undefined direction
- Empty string direction
- Invalid strings (e.g., "diagonal", "45deg", random text)
- Case variations (HORIZONTAL, Horizontal, HoRiZoNtAl)
- Whitespace padding ("  horizontal  ")
- Numbers as strings ("1", "2")

**Action:** Add mutation tests for gradientDirectionFromStore().

### Gap 5: Mode Conditional Rendering Logic
**Current Coverage:** Tests verify that gradient picker appears when mode === "gradient".  
**Missing:**
- What happens when mode is undefined or null?
- What happens when mode is an invalid string?
- What happens when texture param keys don't match the expected `feat_{zone}_texture_*` pattern?
- shouldShowTextureParam edge cases:
  - Key is `feat_body_texture_grad_color_a` but zone is `eye_left` (mismatch)
  - Key includes both `_texture_grad_` and `_texture_spot_` (impossible, but test it)
  - Key is empty string, null, or undefined

**Action:** Add structural mutation tests for mode routing and key matching.

### Gap 6: Color Value Synchronization to Store
**Current Coverage:** Paste button calls onChange, which calls setAnimatedBuildOption.  
**Missing:**
- What if setAnimatedBuildOption throws an error?
- What if the store is in a failed/inconsistent state?
- Multiple rapid pastes (race condition check)
- Multiple pickers updating the same zone simultaneously
- Store missing the slug or animatedBuildOptionValues key (type safety)

**Action:** Add integration tests with store mock failures and concurrent updates.

### Gap 7: Paste Button Hint Auto-clear Timing
**Current Coverage:** Hint is shown on error, disappears after 2 seconds.  
**Missing:**
- What if a second paste happens before the 2-second timeout?
- What if the component unmounts while the timeout is pending? (memory leak)
- Multiple fast pastes (last error wins, or accumulated?)
- Timeout being cleared and restarted (not duplicated)

**Action:** Add timing and concurrency tests for hint lifecycle.

### Gap 8: Shell Script Grep Patterns
**Current Coverage:** test_02c_legacy_picker_scan.sh uses hardcoded keywords.  
**Missing:**
- Typos in legacy picker names that might slip through (e.g., `HEXPickerComponent`, `HexPickercomponent`)
- Keywords as substrings in valid code (e.g., a variable named `legacyColorData` should not match `LegacyColorPicker`)
- Entire word boundary issues (grep -w enforcement)
- Test files and comments that reference legacy code (intentionally excluded?)

**Action:** Add test to run the grep script and verify output format, then add follow-up mutation tests.

### Gap 9: Color Value Round-trip (HexInput → Picker → Store → Display)
**Current Coverage:** Individual components tested in isolation.  
**Missing:**
- E2E: User pastes `#AABBCC` → normalized to `aabbcc` → stored → displayed → changed in picker → stored again
- Type conversion: HexStrControlRow passes string to ColorPickerUniversal, which must treat it as the `color` field
- Format consistency: Does the picker preserve the hex format (uppercase/lowercase)?

**Action:** Add full round-trip integration test.

### Gap 10: File System Stress (if applicable)
**Current Coverage:** No file system operations in color picker code (only clipboard).  
**Missing:** Not applicable for frontend color picker.  
**Note:** If future code adds file upload for ImageMode, this will be in scope.

## Assumptions Encoded in Tests

### A1: Direction default is "horizontal"
If `gradientDirectionFromStore()` receives invalid input, it returns `"horizontal"`. Tests will encode this assumption.

### A2: Hex parsing is case-insensitive
Both `#AABBCC` and `aabbcc` are valid; output normalized to lowercase.

### A3: ZoneTextureBlock gradient picker only appears when mode === "gradient"
Tests will verify that other modes hide the gradient picker entirely.

### A4: Type narrowing is enforced in onChange handlers
If `v.type === "single"`, access only `v.color`. Never assume other fields exist. Tests will use TypeScript strict mode to catch violations.

## Assumptions Validated by Tests

1. **Direction default:** `gradientDirectionFromStore()` returns `"horizontal"` for invalid input — **ENCODED**
2. **Hex case normalization:** Both `#AABBCC` and `aabbcc` normalize to lowercase — **ENCODED**
3. **Mode visibility:** Gradient picker only appears when `mode === "gradient"` — **ENCODED**
4. **Type narrowing:** Type guards are enforced (no access to wrong mode fields) — **ENCODED**
5. **Hint auto-clear:** 2-second timeout is exact and restarts on each paste — **ENCODED**
6. **Concurrent operations:** No built-in debounce (all pastes resolve) — **ENCODED**
7. **Pattern matching:** Substring-based, not word-boundary (permissive) — **ENCODED**

## Risk Exposures Found

1. **Type Union Vulnerability:** Consumers must properly narrow on `v.type` before accessing fields; no runtime validation
2. **Pattern Permissiveness:** `_texture_` and `_color` match without requiring mode prefix (`grad/spot/stripe`)
3. **Case Sensitivity:** Legacy keyword scan is strictly case-sensitive; uppercase variants slip through
4. **Timeout Cleanup:** If component unmounts during paste, timeout may not be cleaned (test verifies it is)
5. **Concurrent Pastes:** All concurrent pastes resolve; last one wins (no built-in deduplication)
6. **Clipboard API Failures:** Missing clipboard API is handled gracefully, but error message may not reach user
7. **Hex Parsing:** No regex validation (simple string checks); allows edge cases like very long strings
8. **Mode/Value Mismatch:** If `lockMode` != `value.type`, behavior is undefined (contract violation)

## Test Determinism & Reproducibility

All tests are deterministic:
- No network calls (mocked clipboard)
- Fake timers used for timeout tests (vi.useFakeTimers/useRealTimers)
- No randomness in test data
- All assertions are stable and repeatable
- Tests can run in any order (no interdependencies)
