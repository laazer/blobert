## Checkpoint

### [M11-08] PLANNING — Same-hit weaken+infect ambiguity
**Would have asked:** If a claw hit drops an enemy from NORMAL to WEAKENED (via HP threshold), should that same hit also trigger the WEAKENED→INFECTED transition?
**Assumption made:** No — infection requires the enemy to already be WEAKENED before the hit lands. A hit that causes the NORMAL→WEAKENED transition does not also infect. This encourages a two-hit pattern (first hit weakens, second infects) and is consistent with the "aggressive play" design intent (short cooldown → quick follow-up).
**Confidence:** Medium

### [M11-08] PLANNING — Attack registration site
**Would have asked:** Where should claw (and future mutation) AttackResources be registered? A dedicated setup function in AttackDatabase, a scene-level _ready() call, or a separate bootstrapper node?
**Assumption made:** Registration happens in a dedicated `_register_defaults()` method on `AttackDatabaseNode` called from `_ready()`. This is the simplest pattern and keeps all base attack definitions colocated. Spec Agent should freeze this.
**Confidence:** Medium
