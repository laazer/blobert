## Notes
- Task 1 (Spec) must complete before Tasks 2 and 3 can begin — spec defines the contract.
- Task 2 (Tests) must complete before Task 3 (Implementation) — tests drive implementation.
- Tasks 3, 4, 5 can be done by the same agent in sequence (Core Simulation Agent) since they're all CI tooling implementation.
- Shadow mode is non-blocking by design (exit 0 regardless of gate result) — this satisfies the "without breaking current CI" acceptance criterion.
- The gate runner is a Python script (not bash) for structured JSON output — consistent with the project's `spec_completeness_check.py` precedent.
- No Python package dependency changes — uses only stdlib (`json`, `dataclasses`, `argparse`, `hashlib`, `pathlib`).
- Tests verify executable behavior (JSON serialization, CLI parsing, file I/O) not spec prose.
