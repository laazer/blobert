## Key Test Design Decisions

1. **Behavioral not documentary**: Tests validate executable behavior (JSON outputs, schema validation, diff logic, clustering algorithms) not markdown prose. All tests use synthetic violation fixtures and assert on runtime results.

2. **INVARIANT_PAIR markers**: Tests TestBaselineDiffDetection::test_new_violation_not_in_baseline_rejected and test_violation_in_baseline_accepted form a pair testing rejection + mutation invariants (new violations without baseline entries must be rejected; grandfathered violations with valid baseline entries must be accepted).

3. **No mock internal state**: Tests avoid mocking internal methods of the audit/baseline modules. Instead, they use real JSON manipulation and dict-based simulation of baseline diff logic to validate behavior at the API boundary.

4. **Fixtures for reproducibility**: Comprehensive fixtures (sample_violations, baseline_template) ensure deterministic test data. All file I/O uses tmp_path for isolation.

5. **Schema validation library**: Tests validate baseline JSON schema compliance using real JSON parsing and field presence checks; compatible with jsonschema library when implemented.
