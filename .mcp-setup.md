### 2. Available MCP Tools for Game Development

#### Analysis Tools
```bash
# Analyze GDScript code patterns (use for script reviews)
POST /mcp/call/analyze_gdscript_project
  └─ Scans GDScript files for patterns, issues, optimization

# Analyze project health (use for metrics and bottlenecks)
POST /mcp/call/get_game_project_health
  └─ Returns performance metrics, code quality scores, test coverage

# Check code quality (use for refactoring decisions)
POST /mcp/call/check_gdscript_security
  └─ Identifies code safety issues, deprecated patterns
```

#### Code Generation Tools
```bash
# Scaffold new GDScript components
POST /mcp/call/scaffold_gdscript_component
  └─ Generates boilerplate for enemies, systems, mechanics
  └─ Parameters: component_type (enemy|system|mechanic), name

# Generate scene templates
POST /mcp/call/scaffold_godot_scene
  └─ Creates .tscn with boilerplate structure
  └─ Useful for: levels, enemy templates, UI panels
```

#### Workflow Tools
```bash
# Run test suite analysis
POST /mcp/call/analyze_test_coverage
  └─ Scans tests/, identifies coverage gaps
  └─ Returns: untested areas, test quality metrics

# Analyze asset pipeline
POST /mcp/call/analyze_blender_pipeline
  └─ Reviews asset_generation/python for issues
  └─ Returns: optimization suggestions, pipeline gaps
```

## Using MCP in Your Development Loop

### Local-First Hybrid Reasoning

The MCP uses hybrid reasoning: **local LLM first, cloud fallback**.

For fast feedback during development:
1. Simple questions → local LLM (instant, no API cost)
2. Complex analysis → cloud LLM (better reasoning, trade off latency)
3. Code generation → cloud preferred (better quality)

You can control via:
```bash
POST /mcp/call/hybrid_reasoning
  {
    "prompt": "Analyze movement_simulation.gd for physics bugs",
    "prefer_local": true,          # Use local LLM if possible
    "complexity": "complex"        # "simple" or "complex"
  }
```

### Cost Optimization

- **Analyze with local LLM**: Review code patterns, suggest refactors
- **Generate with cloud**: Create boilerplate, complex features
- **Test with CI**: Automated validation (no LLM cost)

Expected savings: **70-80% reduction in API costs** vs cloud-only

## Troubleshooting

**MCP tool returns error:**
```bash
# Check tool is registered
curl http://localhost:7891/mcp/tools | jq '.tools | map(.name)'

# Check tool-specific endpoint
curl -X POST http://localhost:7891/mcp/call/[tool_name] -H "Content-Type: application/json" -d '{}'
```

**CodeGraph search returns empty:**
```bash
# Verify index exists
curl http://localhost:7891/api/codegraph/status

# Re-index if needed
curl -X POST http://localhost:7891/api/codegraph/sync
```

**Filesystem browse returns 403:**
```bash
# Path must be within workspace root
# Allowed: /Users/jacobbrandt/workspace/*
# Denied: /Users/jacobbrandt/* (parent), /System/*, etc.
```
